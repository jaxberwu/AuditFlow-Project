# AuditFlow 启动说明

## 🚀 服务已启动

所有三个服务已在后台启动：

1. **Simulator** (Port 5001) - 威胁提供者服务
2. **Engine** (Port 5002) - 审计引擎服务  
3. **Frontend** (Port 5173) - React 前端界面

## ⏱️ 等待服务就绪

首次启动可能需要 30-60 秒，因为需要：
- 初始化数据库
- 创建表结构
- 插入种子数据
- 编译和启动服务

## 🌐 访问地址

### 前端界面（主要入口）
**http://localhost:5173**

### API 端点（用于测试）
- Simulator: http://localhost:5001/threats
- Engine: http://localhost:5002/audit/summary

## ✅ 验证服务是否就绪

在浏览器中访问上述地址，或使用 PowerShell 测试：

```powershell
# 测试 Simulator
Invoke-WebRequest -Uri "http://localhost:5001/threats" -UseBasicParsing

# 测试 Engine
Invoke-WebRequest -Uri "http://localhost:5002/audit/summary" -UseBasicParsing
```

## 🔍 检查服务状态

查看运行中的进程：
```powershell
Get-Process | Where-Object { $_.ProcessName -like "*dotnet*" -or $_.ProcessName -like "*node*" }
```

查看端口占用：
```powershell
netstat -ano | findstr ":5001 :5002 :5173"
```

## 🛑 停止服务

如果需要停止服务，可以：
1. 关闭终端窗口
2. 或使用任务管理器结束 dotnet 和 node 进程

## 📝 预期结果

访问 http://localhost:5173 后，你应该看到：
- **总资产数**: 5
- **不合规设备**: 1 (SN001)
- **合规设备**: 4
- **需要更换的设备列表**: 显示 SN001 的详细信息

## ❓ 如果遇到问题

1. **端口被占用**: 检查是否有其他服务在使用这些端口
2. **数据库错误**: 确保 SQL Server LocalDB 已安装
3. **CORS 错误**: 确保所有三个服务都已启动
4. **页面空白**: 等待服务完全启动（可能需要1-2分钟）
